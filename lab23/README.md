# Lab 23 — ISP & DIP (Smart Home Hub)
## Тема: 
ISP & DIP: рефакторинг і DI через конструктор.
## Мета: 
Застосувати принципи розділення інтерфейсу (ISP) та інверсії залежностей (DIP) для
рефакторингу існуючого коду, а також реалізувати Dependency Injection (DI) через конструктор
для зменшення зв’язаності та покращення тестування.



### Початкова реалізація (Bad)
- `SmartHomeCentralBad`
- Жорстке створення залежностей (`new`)
- Клієнт залежить від усіх підсистем одразу

### Після рефакторингу (Good)
- Вузькі інтерфейси:`ILight`, `IMusicPlayer`, `ISecurityAlarm`
- Клас `SmartHomeCentral` залежить від абстракцій
- Dependency Injection через конструктор
##  Демонстрація роботи
У методі `Main`:
1. Запускається початкова реалізація (з порушеннями)
2. Запускається відрефакторене рішення

Вивід у консоль чітко показує різницю між підходами.
##  Використані принципи
- Interface Segregation Principle (ISP)
- Dependency Inversion Principle (DIP)
- Dependency Injection (Constructor Injection)
##  Висновки
Застосування принципів ISP та DIP дозволило:зменшити зв’язаність між компонентами,підвищити гнучкість та тестованість коду,зробити архітектуру більш розширюваною.
