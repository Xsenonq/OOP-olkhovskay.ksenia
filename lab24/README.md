# Lab 24  
## Strategy + Observer: Динамічна підстановка алгоритмів + тести

###  Мета роботи
Застосувати патерни проєктування Strategy та Observer для створення гнучкої системи обробки числових даних, яка дозволяє: динамічно змінювати алгоритм обробки під час виконання; автоматично сповіщати залежні компоненти про результати обчислень; перевіряти коректність роботи за допомогою юніт-тестів.

## Використані патерни
#  Strategy
###  Призначення
Патерн Strategy дозволяє визначити сімейство алгоритмів, інкапсулювати їх та робити взаємозамінними.

### Реалізація в проєкті
**Інтерфейс:** INumericOperationStrategy

**Метод:** double Execute(double value)

**Конкретні стратегії::**
- SquareOperationStrategy — обчислення квадрата
- CubeOperationStrategy — обчислення куба
- SquareRootOperationStrategy — обчислення квадратного кореня  

 **Контекст:** NumericProcessor
 
Має:
- можливість змінювати стратегію через `SetStrategy()`
- метод `Process()` для виконання поточної операції

### Переваги
- Відповідає принципу Open/Closed
- Легко додати нову операцію без зміни існуючого коду
- Гнучка зміна алгоритму під час виконання

#  Observer

###  Призначення
Патерн Observer дозволяє створити механізм підписки, при якому об'єкти автоматично отримують сповіщення про зміни стану іншого об'єкта.

###  Реалізація в проєкті

**Subject:** ResultPublisher

**Подія:** event Action<double, string> ResultCalculated

**Observers:**
- ConsoleLoggerObserver — вивід у консоль
- HistoryLoggerObserver — збереження історії
- ThresholdNotifierObserver — повідомлення при перевищенні порогу

###  Особливість реалізації
Використано механізм подій C# (event + Action), що є природною реалізацією патерну Observer у .NET.

# Демонстрація роботи
У методі `Main`:
1. Створюється NumericProcessor.
2. Створюється ResultPublisher.
3. Підписуються спостерігачі.
4. Встановлюються різні стратегії.
5. Виконується обробка чисел.
6. Після кожного обчислення викликається PublishResult().
Система демонструє:
- зміну алгоритму під час виконання;
- автоматичне сповіщення всіх підписників;
- збереження історії результатів.

#  Юніт-тестування
Використано xUnit.
Перевірено:
- коректність роботи кожної стратегії;
- правильність зміни стратегії;
- точність результатів обчислень.
