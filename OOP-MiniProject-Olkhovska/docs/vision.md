# Car Rental System

##  Проблема
У компаніях з прокату автомобілів потрібно ефективно контролювати процес оренди:
- які авто доступні
- які вже орендовані
- хто і коли взяв автомобіль

Без системи легко виникають помилки: подвійна оренда, втрата даних або неправильний облік.

##  Цільові користувачі
- Клієнти (орендують автомобілі)
- Працівники компанії прокату (в майбутньому розширенні системи)

## Основні сценарії використання
1. Перегляд доступних автомобілів  
2. Оренда автомобіля клієнтом  
3. Повернення автомобіля в систему  

## Нефункціональні вимоги
- Читабельність коду (розділення на Domain / Application / Infrastructure)
- Тестованість 
- Простота розширення (можливість додати базу даних або API без переписування системи)
- Обробка помилок (захист від некоректних операцій, наприклад повторна оренда авто)

## Обмеження першої ітерації 
- Використовується тільки консольний інтерфейс 
- Відсутня база даних (дані зберігаються в пам’яті через collections)
- Реалізується лише базовий сценарій оренди автомобіля
- Без складної авторизації або ролей користувачів
