Самостійна робота №13 
Тема: Кейси Polly/Retry: короткий звіт. 
Мета: Дослідити та проаналізувати реальні сценарії використання бібліотеки Polly для реалізації політик відмовостійкості (Retry, Circuit Breaker, Timeout) у .NET-застосунках. 
Обрані сценарії

У роботі реалізовано 3 реальні сценарії, де Fail-Safe політики Polly можуть бути особливо корисні:

Нестабільний зовнішній API

Проблема: API може тимчасово повертати помилки (наприклад, HttpRequestException).
Рішення: політика Retry з експоненційною затримкою (WaitAndRetry).
Очікування: після кількох повторних спроб API повертає відповідь.

Тимчасові проблеми підключення до бази даних

Проблема: база може бути перезапущена, заблокована або недоступна кілька секунд.
Рішення: Retry + Circuit Breaker.
Очікування: після кількох помилок Circuit Breaker «відкривається», запобігаючи нескінченним запитам.

Зависла або занадто довга операція

Проблема: операція може тривати надто довго (>2 сек).
Рішення: Timeout + Fallback.
Очікування: при перевищенні ліміту часу повертається запасний результат.
