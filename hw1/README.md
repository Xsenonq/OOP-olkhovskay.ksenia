## Характеристики "God Object" анти-патерну
God Object — це анти-патерн проєктування програмного забезпечення, за якого один клас або об’єкт концентрує в собі надмірну кількість відповідальностей і знань про систему. Такий клас контролює більшу частину логіки застосунку та безпосередньо взаємодіє з багатьма іншими компонентами.
## Основні характеристики God Object:
1.надмірна кількість методів і полів;
2.поєднання бізнес-логіки, роботи з даними та допоміжних функцій;
3.тісна зв’язаність з іншими класами;
складність тестування та підтримки;
4.часті зміни через різні причини;
Використання God Object призводить до порушення принципів SOLID, зокрема SRP.
## Порушення принципу SRP
Single Responsibility Principle (SRP) стверджує:клас повинен мати лише одну причину для зміни.
## Приклад класу, що порушує SRP
```csharp
public class UserManager
{
    public void CreateUser(string name)
    {
        Console.WriteLine($"User {name} created");
    }

    public void SaveUserToDatabase(string name)
    {
        Console.WriteLine($"User {name} saved to database");
    }

    public void SendWelcomeEmail(string name)
    {
        Console.WriteLine($"Welcome email sent to {name}");
    }
}

## Порушення SRP
Клас UserManager має декілька відповідальностей-
1.Створення користувача (бізнес-логіка).
2.Збереження даних у базі даних.
3.Надсилання email-повідомлень.
Кожна з цих відповідальностей може змінюватися з незалежних причин (зміна БД, зміна email-сервісу, зміна бізнес-логіки). Отже, клас має більше ніж одну причину для зміни, що є прямим порушенням SRP та типовим прикладом God Object.
## Дотримання SRP
Для дотримання принципу SRP необхідно розділити відповідальності між окремими класами
```csharp
public class UserService
{
    public void CreateUser(string name)
    {
        Console.WriteLine($"User {name} created");
    }
}

public class UserRepository
{
    public void Save(string name)
    {
        Console.WriteLine($"User {name} saved to database");
    }
}
public class EmailService
{
    public void SendWelcomeEmail(string name)
    {
        Console.WriteLine($"Welcome email sent to {name}");
    }
}

## Переваги
1.кожен клас має одну чітку відповідальність;
2.код стає більш читабельним і підтримуваним;
3.спрощується тестування;
4.зменшується зв’язаність між компонентами;
5.система легше масштабується.

## Висновок
Анти-патерн God Object є поширеною помилкою початкового проєктування, яка призводить до складного та нестабільного коду. Дотримання принципу SRP дозволяє створювати модульні, зрозумілі та гнучкі системи, що значно полегшує подальший розвиток програмного забезпечення.
