## Лабораторна робота №21
OCP: гнучкі алгоритми розрахунку (Strategy, Factory Method)
## Мета роботи
Застосувати принцип відкритості/закритості (Open/Closed Principle) для створення гнучкої системи розрахунку вартості доставки з використанням патернів Strategy та Factory Method.
## Короткі теоретичні відомості
OCP передбачає, що програмні модулі повинні бути відкритими для розширення, але закритими для зміни.
Strategy дозволяє інкапсулювати різні алгоритми та підміняти їх під час виконання програми.
Factory Method відповідає за створення об’єктів стратегій без прив’язки до конкретних реалізацій.
## Реалізація
Для алгоритмів доставки створено інтерфейс IShippingStrategy.
Кожен тип доставки реалізовано в окремому класі-стратегії.
Клас DeliveryService працює лише з інтерфейсом стратегії та не залежить від конкретних реалізацій.
Клас ShippingStrategyFactory створює відповідну стратегію на основі введеного типу доставки.
## Демонстрація OCP
Додано нову стратегію NightShippingStrategy без зміни логіки класу DeliveryService, що підтверджує дотримання принципу відкритості/закритості.
## Висновки
У роботі реалізовано гнучку систему розрахунку доставки з використанням патернів Strategy та Factory Method. Система є розширюваною та відповідає принципу OCP.
